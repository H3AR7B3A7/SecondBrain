# Retro
A good Retrospective is an important part of [[Agile Project Management]].

## Structure
### Intro
- Go over last retro action points
- ENERGIZER

### Mid
- Collect inputs (mad / sad / glad)
- Group inputs
- Filter & clear up inputs
- Discuss inputs
	- Vote
	- Question the topic
	- Create action points

### End
- Next person to do retro
- Review the retro
